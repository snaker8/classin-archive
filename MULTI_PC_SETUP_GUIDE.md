# 멀티 PC 자동 업로드 설정 가이드

다른 PC에서도 자동 업로드 기능을 사용하려면 다음 설정 과정을 **반드시** 거쳐야 합니다. 단순 파일 복사만으로는 작동하지 않는 경우가 많습니다.

## 필수 준비물
1. **Node.js**: [Node.js 공식 홈페이지](https://nodejs.org/)에서 LTS 버전을 다운로드 및 설치해야 합니다.
2. **.env.local 파일**: 중요 보안 키가 들어있는 파일로, 작동 중인 메인 PC에서 복사해와야 합니다.

---

## 설정 단계

### 1. 전용 설치 폴더 사용 (보안 강화)
전체 프로젝트 소스 코드를 복사할 필요 없이, 실행에 꼭 필요한 파일만 모아둔 `dist-monitor` 폴더를 사용합니다.

1. **메인 PC**의 `classin-archive` 폴더 안에 새로 생성된 **`dist-monitor`** 폴더를 찾습니다.
(이 폴더에는 핵심 프로그램과 필수 보안 키(`env.local`)만 들어있어 훨씬 가볍고 안전합니다.)

2. 이 `dist-monitor` 폴더를 통째로 복사해서 **새로운 PC**의 원하는 위치(예: `C:\classin-monitor`)에 붙여넣습니다.

### 2. 프로그램 설치 (1회만 수행)
새로운 PC의 `dist-monitor` 폴더 주소창에 `cmd`를 치고 엔터를 눌러 검은색 창을 엽니다.
그 후 아래 명령어를 입력하고 엔터를 누릅니다:

```bash
npm install
```
(필요한 라이브러리가 자동으로 설치됩니다.)

### 3. 자동 실행 등록 (선택)
PC가 켜질 때마다 자동으로 감시가 시작되게 하려면:
**`install-startup.bat`** 파일을 더블 클릭하여 실행합니다.

### 4. 감시 폴더 설정
처음 실행하면 어떤 폴더를 감시할지 물어봅니다. ClassIn이 저장되는 폴더 경로를 입력해주면 됩니다.
(이미 설정되어 있다면 `monitor-config.json` 파일을 수정하거나 삭제하여 재설정할 수 있습니다.)

---

## 작동 확인 방법

1. 새로운 PC에서 **`run-monitor.bat`**를 실행해 봅니다.
2. 검은색 창이 뜨고 파일 감시가 시작되는지 확인합니다.
3. 에러 메시지가 뜬다면 내용을 확인해주세요.

> **주의:** `.env.local` 파일은 업로드 권한을 가진 중요한 키를 포함하므로, 이 폴더가 외부로 유출되지 않도록 주의해 주세요.
